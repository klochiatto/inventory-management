System Architecture

Factory Inventory Management System

A full-stack demo application for inventory tracking, order management, demand forecasting, and spending analytics across factory operations. Single-page frontend backed by a stateless REST API over in-memory mock data.

Frontend Vue 3 + Vite · :3000 Backend FastAPI · :8001 Data JSON files (in-memory) Localization EN / JA

Tech Stack

Frontend Port 3000

Single-page application
  • Vue 3 — Composition API
  • Vite — dev server & build
  • vue-router — client-side routing
  • axios — HTTP client

Backend Port 8001

REST API
  • FastAPI — routing & OpenAPI docs
  • Uvicorn — ASGI server
  • Pydantic — response validation
  • CORS — open to all origins

Data In-memory

No database
  • JSON files in server/data/
  • Loaded once via mock_data.py
  • Filtered per-request in Python
  • Entities: inventory, orders, demand, backlog, spending

Data Flow — a filtered request

1 User sets filters FilterBar → useFilters refs (warehouse, category, status, month)
2 View calls api.js filters serialized into query-string params
3 FastAPI endpoint apply_filters() + filter_by_month() over in-memory lists
4 Pydantic validates typed models shape the JSON response
5 View renders data in refs → computed props → template
Response returns along the same path: JSON → axios → Vue refs → cached computed properties → reactive DOM update

Reactivity pattern: raw API data lives in refs (e.g. allOrders, inventoryItems); all derived values — totals, charts, filtered lists — are computed properties that recalculate only when their dependencies change.
